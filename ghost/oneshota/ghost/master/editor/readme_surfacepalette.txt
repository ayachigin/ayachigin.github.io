サーフェスパレットの使い方

サーフェスパレットは、シェルの各サーフェスのサムネイルを表示する機能です。
普通にシェルを読み込んだだけだと、サーフェスのはしっこしか表示されず、きちんと確認できないかもしれません。
そういうときは、シェルのフォルダにある設定ファイル「surfaces.txt」に、さとりすと用の設定を書き込みます。

「さとりすとwiki」のほうが画像付きでわかりやすいかもしれません。
http://wikiwiki.jp/satolist/?%B3%C6%A5%A6%A5%A4%A5%F3%A5%C9%A5%A6%A4%CE%B2%F2%C0%E2%2F%A5%B5%A1%BC%A5%D5%A5%A7%A5%B9%A5%D1%A5%EC%A5%C3%A5%C8


■表示位置をずらす
単純に表示位置をずらしたい場合は、surfaces.txtの上の方（中括弧"{"があるところより上の行）に、

//satolist.palette.offset,X座標,Y座標

と書きます。
コメントアウトで無効化されているように見えますが、これはさとりすとでは正しく読み込まれ、SSPからは無視されるための対策です。
必ず//からはじめてください。
また、この記述を無効化したい場合は、///（３つ）ではじめてください。

例:
//satolist.palette.offset,50,60

この場合、左に50、上に60、サーフェスをずらします。

サブキャラクタや別のポーズなどでずらす場所を個別に指定したい場合は、
そのサーフェスのブレスにこの設定を書き込みます。

例:
surface0
{
//satolist.palette.offset,20,30
…（通常のsurfaces.txtの記述）
…
}

この場合、surface0に限り、ずらす量が 20,30 になります。


■表示を引き伸ばす
差分の位置が小さかったりなどで、サーフェスパレットから見づらい場合などのために、
サーフェスパレットに拡大縮小を設定することができます。

「表示位置をずらす」と同様にサーフェスブレス、もしくはファイルの上の方に、

//satolist.palette.expand,倍率

と書きます。

例:
//satolist.palette.expand,1.5

この場合、サーフェスを右下の方向に1.5倍引き伸ばして表示します。

■表示する・しないを設定する
表情数が多いゴーストなどで、表示するサーフェス数を限定したい場合などに、
サーフェスパレット・サーフェスビューワに表示非表示を設定することができます。
差分の位置が小さかったりなどで、サーフェスパレットから見づらい場合などのために、

「表示位置をずらす」と同様にサーフェスブレス、もしくはファイルの上の方に、

//satolist.palette.visible,0
または
//satolist.palette.visible,1

と書きます。

0なら非表示、1なら表示になります。
何も指定しない場合は、全体に表示を設定した状態になります。
サーフェスビューワの表示を変更する場合は、
//satolist.viewer.visible,0
//satolist.viewer.visible,1
を使用します。


■表示枠の大きさを変更する
サーフェスの量が大きいために枠を小さくしたい、見やすいように枠を大きくしたい場合などのために、
サーフェスパレットに枠の大きさを設定することができます。

surfaces.txt の上の方に次の記述をします。
この設定は、サーフェスブレス内で設定することが できません。

//satolist.palette.frame,幅,高さ

例:
//satolist.palette.frame,200,200

この場合、サーフェスの表示枠を200x200で表示します。
何も設定しない場合の表示枠の大きさは、100x100です。

■着せ替えリスト表示用のキャラクターID（スコープ）を指定する
サーフェスごとに使用する着せ替え定義のスコープを指定できます。
この設定は、サーフェスブレス内でのみ設定することができます。

//satolist.scope,(スコープID)

(スコープID)は半角数値で設定します。
sakura側なら0、kero側なら1。2以降も使えます。

例:

surface0-9
{
//satolist.scope,0
}

surface10-19
{
//satolist.scope,1
}

こうすると0〜９のサーフェスではsakura側のきせかえが、１０〜１９ではkero側のきせかえが適用されます。
この機能はサーフェスビューワがのオプションにある「surfaces.txt によるスコープ指定を優先」が有効の時にのみ機能します。


■静止画で表示するアニメーションパターンを指定する
アニメーション再生が無効なときに、表示しておくアニメーションパターンを設定できます。
この機能を使うとサーフェスパレットやサーフェスビューワがアニメーションしないときに、
目を書いていないベースサーフェスにアニメーションの目を合成するなどしているために、目が表示されない問題を解決できます。
この設定は、サーフェスブレス内でのみ設定することができます。

//satolist.surface.default,(アニメーションID),(パターンID)

例:
surface0
{
animation0.pattern0,（略）

//satolist.surface.default,0,0
}

こうすると、animation0.pattern0（0pattern0）のアニメーションパターンを静画で表示するようになります。
同じサーフェス内で同じアニメーションIDに指定できるパターンはひとつです。
